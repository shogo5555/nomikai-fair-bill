## プロダクトの目的

スマホで30秒以内に使える「不公平感の少ない飲み会会計ツール」。
参加者ごとに係数（倍率）を設定し、合計金額を比例配分する。外部サービス不要、ブラウザ完結。

## 技術スタック

- **フレームワーク**: React + Vite (SPA)
- **言語**: TypeScript
- **スタイル**: CSS Modules または Tailwind CSS（未決定時はCSS Modules）
- **状態管理**: React 組み込み（useState / useReducer）のみ。外部ライブラリ不要
- **バックエンド**: なし。全ロジックはブラウザ内で完結

## MVPの範囲

1. 合計金額の入力
2. 参加者の人数・名前・係数（倍率）の入力
3. 端数処理の選択（切り捨て / 切り上げ / 四捨五入）
4. 一人あたりの支払い金額を一覧表示

UIはステップ式ウィザード（合計入力 → 人数入力 → 係数設定 → 結果表示）で構成する。

## 計算ルール

倍率ベースの比例配分：

```
各人の支払い額 = 合計金額 × (その人の係数 / 全員の係数の合計)
```

- 係数はデフォルト `1.0`（正の実数）
- 全員の係数が `1.0` の場合、結果は均等割りと一致する（均等割り専用モードは作らない）
- 端数は選択した処理方式で丸め、差額は最後の1人に集約する

## UI方針

- **モバイルファースト**。画面幅 390px 基準で設計
- **ステップ式**: 一度に1ステップだけ表示し、戻る／次へで遷移
- 入力は最小限。名前は省略可能（「参加者1」などデフォルト値を使う）
- 結果画面はコピーしやすいテキスト形式でも表示する

## 実装後の検証コマンド

```bash
npm install          # 依存パッケージのインストール
npm run dev          # 開発サーバー起動（Vite HMR）
npm run build        # プロダクションビルド（dist/ に出力）
npm run preview      # dist/ をローカルでプレビュー
npm run lint         # ESLint
npm test             # ユニットテスト（Vitest 想定）
```

計算ロジックは純粋関数として切り出し、Vitest でユニットテストを書く。

## やらないこと（MVPスコープ外）

- ログイン・アカウント機能・履歴保存
- バックエンド・データベース・外部API
- PayPay / 外部決済サービスとの連携
- 広告・店舗予約連携
- 均等割り専用モード（全員1.0で自然に均等割りになるため不要）

## アーキテクチャ

```
src/
  components/   # ステップごとのUIコンポーネント
  hooks/        # カスタムフック（ウィザード状態管理など）
  utils/        # 計算ロジック（純粋関数）← テスト対象
  types/        # 型定義
```

計算ロジックは `utils/` に純粋関数として分離し、UIコンポーネントと混在させない。
